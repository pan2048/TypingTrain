The size of an integer can be arch. This means the size of the data type will be derived from the architecture of the machine. An integer the size of which is arch will be 32 bits on an x86 machine and 64 bits on an x64 machine. An arch integer is primarily used when indexing some sort of collection.
Each signed variant can store numbers from -(2^(n-1) to 2^(n-1) -1, where n is the number of bits that variant uses. For example, i8 can store numbers from -(2^7) to 2^7 -1 âˆ’ here we replaced n with 8.
Each unsigned variant can store numbers from 0 to (2^n)-1. For example, u8 can store numbers from 0 to 2^7, which is equal to 0 to 255.
An integer overflow occurs when the value assigned to an integer variable exceeds the Rust defined range for the data type. On execution, data overflow will return a warning. The overflow values will be truncated. 
```
   let result = 10;    // i32 by default
   let age:u32 = 20;
   let sum:i32 = 30;
   let mark:isize = 10;
   let count:usize = 30;
```
